Regularization

Regularization is used to lower the weight: because high weight means more importance, if too many unit of weight are high => over-fitting regularization used by add the penalize to the loss

From above formula, the norm of the weight (length or size of each vector in the weight) is calculated ⇒ If the weight is high then the regularization is high ⇒ Loss will be high.

Why regularization helps?

When loss is high ⇒ the weight tends to lower to 0.0.

WHY?

Because of Gradient Descent

θ\thetaθ﻿ is represented to the weight. And f()f()f()﻿ is the loss function. When the loss is too large then the weight minus by this value will go to 0.0

If some of the layers are modified close to 0.0 then the model will be less complex ⇒ reduce variance ⇒ prevent over-fitting.

Disadvantage

When λ\lambdaλ﻿ is too large ⇒ Loss is too large ⇒ The weight will trend to 0.0 ⇒ many unit of weight will decay to 0.0

L1 reduces weight -> less importance weight may to 0.0 (WHY?) => works well for feature selection => result in sparse network => L2 L2

As the image describes, the L2 regularization represents the orange and the loss function represent the blue one. When L2 is applied, the loss will be upper (a)(a)(a)﻿. The light blue area next to the orange circle area is the lowest loss we can achieve.

Why do we do that?

Practically, we don’t want model to come to the lowest loss, that means over-fitting because the model learn all the features from train data and will be confused when meeting the test data that is slightly different from the train set. We just want the model to be near the lowest point.

Back to the question why L2 is better than L1, at specific situation as image below depicts, the loss function intersect with L1 regularization at w∗w^*w∗﻿ which means w2=ww_2 = ww2​=w﻿ and w1=0w_1 = 0w1​=0﻿. This may lead to sparsity in weights ⇒ It will cause vanishing gradient. The L1 tends to choose the acute angle and usually leads to this problem than L2.

Code

# set l1_weight and l2_weight to non-zero values to enable penalties

# inside the training loop (given input x and target y)

pred = net(x)
loss = loss_func(pred, y)

# compute penalty only for net.hidden parameters
l1_penalty = l1_weight * sum([p.abs().sum() for p in net.hidden.parameters()])
l2_penalty = l2_weight * sum([(p**2).sum() for p in net.hidden.parameters()])
loss_with_penalty = loss + l1_penalty + l2_penalty

optimizer.zero_grad()
loss_with_penalty.backward()
optimizer.step()

# The pre-penalty loss is the one we ultimately care about
print('loss:', loss.item())
