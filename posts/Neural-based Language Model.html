Neural-based Language Model

The difference between Neural-based LM and N-Grams LM

Advantages:

Don't need smoothing.

Can handle much longer histories.

Can generalize over context of similar words.

Has much higher accuracy than N-Grams.

Used for Machine Translation, Dialog, Language Generation.

Disadvantages:

Strikingly slower to train than traditional LM (N-Grams).

Many tasks an N-Grams in still the right tool.

Feed Forward Language Model

Due to the large length of sequences and according to Markov:

⇒ P(wt∣w1,w2,...,wt−1)≈P(wt∣wt−N+1,...,wt−1)P(w_t|w_1,w_2,...,w_{t-1})\approx P(w_t|w_{t-N+1},...,w_{t-1})P(wt​∣w1​,w2​,...,wt−1​)≈P(wt​∣wt−N+1​,...,wt−1​)﻿

E.g: 4-Grams example:

P(w4∣w1,w2,w3)P(w_4|w_1,w_2,w_3)P(w4​∣w1​,w2​,w3​)﻿

However, in neural network language model, we don't use word for unit, we use embeddings of words.

Embedding

In neural network language model, the prior context represent by embedding (vector) than exact words in N-Grams Language Model.

⇒ Allow neural network LM to generalize to unseen data much better than N-Grams.

E.g: In training set we see this sentence:

I have to make sure that the cat gets fed.

However, in training set we don't have gets fed after dog. In our test set, there is a sentence:

I forgot to make sure the dog gets.

In neural network LM, we have chance to know cat embeddings is similar to dog embeddings so that the model can predict word fed even after seeing dog.

Model Architecture

Model above uses Word2Vec for embedding, this is called Pretraining. We can also learn embeddings simultaneously with training network. This often used for tasks that have strong constraints on what makes a good representation.

To learn embeddings,

  • Firstly, we add extra layer at the start of the model, then propagate the error all the way back to that extra layer.
  • Start the embeddings with random values.
  • For input layer, we represent each of N words as one hot vector.
