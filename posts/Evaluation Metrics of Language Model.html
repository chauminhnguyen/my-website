Evaluation Metrics of Language Model

Most Language Models (LM) estimate the probability as product of each symbol (character/word/sub-word) 's probability given by its preceding symbols:

P(w1,w2,...,wn)=p(w1)∗p(w2∣w1)∗p(w3∣w1,w2)∗...∗p(wn∣w1,w2,...,wn)P(w_1,w_2,...,w_n)=p(w_1)*p(w_2|w_1)*p(w_3|w_1,w_2)*...*p(w_n|w_1,w_2,...,w_n)P(w1​,w2​,...,wn​)=p(w1​)∗p(w2​∣w1​)∗p(w3​∣w1​,w2​)∗...∗p(wn​∣w1​,w2​,...,wn​)﻿

To measure the performance of LM, we use Perplexity, Cross Entropy or Bits-Per-Character/Word (BPC / BCW).

Cross Entropy

Again we turn back to the story about Bob and Alice, Alice is also an animal lover. However, she love to talk about cat and the frequency of 4 word about animals she talk about will be:

At first she use Bob's code to send him an message, but it seems like her message is longer then their expectation, that is because Bob's code is optimal for Bob's probability. The average optimal length of Bob when he use his code is 1.75 bits, when Alice's is 2.25 bits.

This length – the average length of communicating an event from one distribution with the optimal code for another distribution – is called the Cross Entropy. Formally, we can define cross-entropy as:

Hp(q)=∑xq(x)log⁡2(p(x))H_p(q)=\sum_xq(x)\log_2(p(x))Hp​(q)=∑x​q(x)log2​(p(x))﻿

In this case, it's the Cross Entropy of Alice's word frequency with respect to Bob's word frequency.

If Alice want to optimize her message she need to build her own code. If Bob accidentally use Alice code he will costcostcost﻿ more than Alice due to the the average optimal length is 2.375 bits. We will have 4 possibilities:

  • Bob using his own code: H(p)=1.75H(p)=1.75H(p)=1.75﻿ bits
  • Alice using Bob’s code: Hp(q)=2.25H_p(q)=2.25Hp​(q)=2.25﻿ bits
  • Alice using her own code: H(q)=1.75H(q)=1.75H(q)=1.75﻿ bits
  • Bob using Alice’s code: Hp(q)=2.375H_p(q)=2.375Hp​(q)=2.375﻿ bits

We can see that Hq(p)≠Hp(q)H_q(p)\neq H_p(q)Hq​(p)=Hp​(q)﻿, we can see that the area of Hq(p)H_q(p)Hq​(p)﻿ is larger than Hp(q)H_p(q)Hp​(q)﻿. Moreover, the average optimal length between them is not equal

Cross Entropy shows how difference two probability distributions are. The more these distributions are, the bigger the value of the Cross Entropy of these distributions are.

Perplexity

Perplexity measures the uncertainty.

The perplexity measures the amount of “randomness” in our model. If the perplexity is 3 (per word) then that means the model had a 1-in-3 chance of guessing (on average) the next word in the text.

PPLp(q)=2Hp(q)PPL_p(q)=2^{H_p(q)}PPLp​(q)=2Hp​(q)﻿

Ex: LM with entropy of 3 bits.

⇒ That LM has to choose among 23=82^3=823=8﻿ possible options.

⇒ That LM has perplexity of 8.

For this reason, it is sometimes called the average branching factor.

Use Entropy as metrics

Metrics use in machine learning have their bounds. For example, in classification, the best value will be 100% for accuracy or 0 for mean squared error. When model's accuracy reach 99%, we know that that model is good.

When LM has a cross entropy of 7, we don't know how far it is from the best value if we don't know the bounds, this can be challenges for us to improve the model.

Mathematical Bounds

We all know that the smallest entropy is zero. But the entropy of the LM can only equal to zero if that LM has 1 symbol:

min⁡H(p)=−1∗log⁡(1)=0\min{H(p)}=-1*\log(1)=0minH(p)=−1∗log(1)=0﻿

If language has 2 symbols an it is uniform distribution, its entropy would be:

H(p)=−0.5∗log⁡2(0.5)−0.5∗log⁡2(0.5)=1H(p)=-0.5*\log_2(0.5)-0.5*\log_2(0.5)=1H(p)=−0.5∗log2​(0.5)−0.5∗log2​(0.5)=1﻿

⇒ Entropy of probability distribution is maximal when it is uniform.

⇒ LM that has maximal entropy is the one in which all the symbols appear with equal probability.
